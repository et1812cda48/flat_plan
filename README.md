## Этапы паплайна:
1. Обработка изображения схемы квартиры и наложение различных фильтров.
2. Извлечение координат из схемы.
3. Сохранение координат в файл json.

## В проекте был использован только технология OpeCV, ввиду ограниченности по времени, поскольку обучение моделей нейронных сетей довольно ресурсоемкий процесс.

## Слабые места проекта:
1. Чувствительность к качеству изображения: Шумы, неоднородное освещение ухудшают результаты.
2. Проблемы с тонкими стенами: Алгоритм Canny может не находить слабовыраженные границы.
3. Объединение близких линий: Отсутствует логика объединения коллинеарных сегментов.
4. Только ортогональные стены: Алгоритм предполагает, что стены горизонтальные или вертикальные.
5. Нет распознавания элементов: Двери, окна, проемы не детектируются.
6. Отсутствие семантики: Не различаются наружные/внутренние стены, несущие конструкции.
7. Дублирование линий: Одна стена может быть представлена несколькими сегментами.
8. Нет векторизации: Результат в пикселях, без привязки к реальным размерам.
9. Ограниченная валидация: Нет проверки топологической корректности плана

## Улучшения для следующих итераций:
1. Улучшение качества модели.
2. Детекция дверей и окон.
3. OCR для извлечения размеров.
